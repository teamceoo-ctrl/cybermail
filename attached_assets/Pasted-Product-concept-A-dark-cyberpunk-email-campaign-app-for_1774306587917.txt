Example blacklist alert

Warning: Sending IP reputation issue detected
Current outbound IP appears on one or more reputation lists or has elevated complaint/bounce signals.
Recommended action: pause active campaigns, review recent bounce patterns, and switch to a verified sending provider until resolved.

Safe architecture
Frontend: Next.js + Tailwind
Backend: Node.js / NestJS
Queue: Redis + BullMQ
Database: PostgreSQL
Email: SES, Postmark, Mailgun, SendGrid, Resend, or user-owned SMTP
Templates: React Email or MJML
Logs: webhook ingestion for bounces, complaints, deliveries
UI prompt
Design a web-based email campaign platform with a dark hacker-inspired cyberpunk interface. Use black and charcoal backgrounds, neon green accents, terminal-like panels, glowing borders, monospace typography, and subtle animated scanline effects.

The product should be for legitimate consent-based email campaigns only.

Include these pages:
1. Dashboard with campaign metrics, bounce rate, complaint rate, unsubscribe rate, recent activity, and a reputation monitor widget
2. Contacts page with CSV import, tagging, segmentation, and custom fields
3. Template editor with merge tags like {{first_name}}, {{email}}, and {{company}}
4. SMTP Profiles page with manual SMTP configuration, connection testing, encryption options, and sending limits
5. Campaign Composer with subject line, preview text, recipient segment, template preview, test send, and scheduling
6. Delivery Logs page with statuses: queued, sent, delivered, bounced, complained, unsubscribed, failed
7. Reputation Monitor page showing domain auth status, blacklist/reputation alerts, and remediation suggestions

Design requirements:
- responsive SaaS admin layout
- left sidebar navigation
- neon green buttons and status lights
- terminal-style data tables
- clean, modern UX
- glowing alert banners for risk notifications
- professional but edgy cybersecurity aesthetic
